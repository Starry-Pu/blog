# 👨‍💻 关于我 (About Me)

> "机械转码第 5 年。比起画图纸，操控数控机床，我更喜欢写代码这种能立马看到结果的感觉。"

你好，我是 **Pu Starry**。
以前学的是机械制造，现在是一名**前端开发**。

---

## 🔧 为什么转行？

大学我是学**机械**的。每天对着齿轮、轴承和 CAD 图纸，感觉反馈周期太长了——画好的图纸变成零件，可能要等好几周。
实习的时候接触到了前端，发现改一行代码，浏览器刷新一下就能看到效果，这种**“即时反馈”**的感觉我很喜欢，于是就开始自学转行了。

虽然不是科班出身，但以前搞机械练出来的逻辑思维，对写代码帮助挺大的——毕竟本质上都是把零件组装起来，让机器（程序）跑通。

---

## 💻 我怎么写代码

我比较实用主义，工具能解决问题就行，不纠结“最好的语言”。

* **Vue**：
    用了好几年，也是工作主力。像之前的**在线教育平台**和**官网优化**，都是用 Vue 搞定的，上手快，干活效率高。
    
* **React / Taro**：
    最近在写**“蘑菇宅急便”**小程序，为了学点新东西，特意选了 React。说实话，一开始用 `useEffect` 确实容易晕，但写多了也觉得挺有意思的，能换个角度看问题。

* **喜欢“偷懒”**：
    我不喜欢做重复的事。如果一个功能要写三遍，我肯定会把它封装成函数或者 npm 包。现在的博客部署也是用 GitHub Actions 自动化的，写完推送就不用管了，省事。
